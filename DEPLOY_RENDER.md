# Deploy LegalLens to Render

Complete deployment guide for both frontend and backend on Render.

## 🚀 Quick Deploy (Automated)

1. **Push to GitHub** (already done ✅)

2. **Go to Render Dashboard**: https://dashboard.render.com

3. **Click "New +"** → Select **"Blueprint"**

4. **Connect Repository**: Choose `LegalLens` from your GitHub

5. **Render will auto-detect** the `render.yaml` file and create:
   - PostgreSQL database (legallens-db)
   - Backend API (legallens-backend)
   - Frontend app (legallens-frontend)

6. **Click "Apply"** - Render will:
   - Create PostgreSQL database
   - Build and deploy backend (with auto migrations)
   - Build and deploy frontend
   - Connect everything automatically!

## 🔧 Manual Configuration (Optional)

If you want to customize beyond the blueprint:

### Backend Environment Variables
```
DATABASE_URL=<auto-set-from-database>
SECRET_KEY=<auto-generated>
APP_NAME=LegalLens
DEBUG=false
LLM_PROVIDER=local
STORAGE_PROVIDER=local
CORS_ORIGINS=["https://legallens-frontend.onrender.com","http://localhost:3000"]
PYTHON_VERSION=3.11.0
```

### Frontend Environment Variables
```
NEXT_PUBLIC_API_URL=https://legallens-backend.onrender.com
NEXT_PUBLIC_APP_NAME=LegalLens
NODE_VERSION=18.17.0
```

## 🐛 Troubleshooting

### Backend Build Failed
**Check logs**: Look for Python errors
**Common fixes**:
- Ensure all requirements are in requirements.txt
- Check Python version (3.11 recommended)
- Verify DATABASE_URL is set

### Frontend Build Failed
**Check logs**: Look for npm errors
**Common fixes**:
- Ensure NEXT_PUBLIC_API_URL is set
- Clear build cache and redeploy
- Check Node version (18.x recommended)

### Database Connection Error
**Fix**: Ensure DATABASE_URL is linked from database
**Check**: Backend logs for connection errors

### CORS Errors
**Fix**: Update CORS_ORIGINS in backend env vars
**Format**: `["https://your-frontend.onrender.com"]`

### 404 on API Calls
**Check**: NEXT_PUBLIC_API_URL in frontend matches backend URL
**Fix**: Update frontend env var and redeploy
